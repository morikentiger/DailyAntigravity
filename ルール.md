# DailyAG 運用ルール

このリポジトリは Antigravity を「日次生成記録装置」として固定化するためのものです。以下のルールを厳守してください。

## 1. フォルダ構成ルール
成果物は「日付ベースの親フォルダ」と、その中の「連番付きアクティビティフォルダ」で構成します。
### 1. フォルダ構造とデプロイ
- **プロジェクトルート**: GitHub Pages 用の `index.html` (ポータルサイト) を配置。
- **日別フォルダ**: `DayX__YYYYMMDD/` 形式で作成。各日のポータルからリンクする。
- **アクティビティフォルダ**: 日別フォルダ内に `N_内容/` 形式で作成。`Log.md` と `media/` を含む。

**親フォルダ名形式**: `DayX__YYYYMMDD`
- 例: `Day0__20260115`

**子フォルダ名形式**: `N_内容` (Nは1からの連番)
- 例: `1_Setup`, `2_Integration`, `3_Refinement`, `4_MobileReady`

**各フォルダ内構成**:
- `DayX__YYYYMMDD/` (親フォルダ)
    - `index.html`, `style.css`, `app.js`: その日のダッシュボード（閲覧用本体）
    - `N_内容/` (子フォルダ): 個別のアクティビティ
        - `Log.md`: 観測ログ
        - `media/`: 成果物メディア
        - `(成果物)`: ソースコード等

## 2. ログテンプレート（DayX-X.md）
ログファイルは以下のセクションを必ず含めてください。

```markdown
# Day X (または内容に応じたID)

## POST
（SNS投稿用。2〜4行。成果の要約）

## LOG
- 投げたもの：
- 観測：
- 一言：

## ARTIFACTS
- 動画：
- スクリーンショット：
- 備考：

OK_TO_POST: yes または no
```

## 3. 自動投稿パイプライン（GitHub Actions）
- `DailyAG/**/*.md` が更新されると GitHub Actions が起動します。
- `OK_TO_POST: yes` の場合、`## POST` セクションの内容が SNS（Bluesky等）に自動投稿されます。
- `ARTIFACTS` に動画があれば動画を優先、なければスクショを添付します。

## 4. メディア記録の優先順位
1. **動画**: 挙動がわかるものを最優先。
2. **スクリーンショット**: 動画が撮れない場合の補助。
3. **テキストのみ**: 何も生成できなかった場合。

## 5. Antigravity への接し方
- 創作を求めない。記録装置として扱う。
- クオリティを求めない。失敗も「No Output」として記録する。
- 毎日淡々と生成させ、フォルダを置くことで完了とする。

## 6. タスク管理ルール（やりたいリスト.md）
開発の開始と終了時に必ず `やりたいリスト.md` を更新し、進捗を同期させてください。

### 開発開始時
- `## 📅 ロードマップ（今後やること）` セクションの対象タスクを `[ ]` から `[/]` (着手) または `[x]` (完了) に変更する前に、現在の状態を正しく把握する。
- RPAが自動指示を出す際のソースとなるため、重複や漏れがないようにする。

### 開発完了時
- `## 📅 ロードマップ（今後やること）` のタスクを `[x]` にし、完了日と絵文字を添える。
- `## ✅ やったこと（文明の記録）` セクションに、その日のミッション名と主要な成果（技術的進歩など）を追記する。
- `やりたいリスト.md` と `checkpoint.md` の状態を一致させる。
